# docs/ — Team Decisions & Plans

This directory contains **our team's distilled thinking** — the decisions, architecture, and strategy we've converged on. These are living documents that evolve as we build.

## Files

| File | Purpose | Status |
|------|---------|--------|
| **PRD.md** | Master product requirements — features, priorities, demo script, Q&A prep | Primary reference |
| **CONCEPT.md** | The "what and why" — EDEN's story, 4-layer architecture, K8s mapping, Syngenta integration | Pitch foundation |
| **ARCHITECTURE.md** | Technical architecture — data flow, DynamoDB schema, AgentCore setup, nutritional model | Build reference |
| **RISKS-AND-GAPS.md** | Risks, gaps, judge psychology, scope control, build/fake/cut decisions | Execution guide |

## Relationship to other directories

- **`reference/`** = raw inputs (challenge brief, slides, APIs, organizer materials) — what we were GIVEN
- **`docs/`** = distilled outputs (our decisions, architecture, plan) — what we DECIDED
- **`brainstorm/`** = creative process (idea evolution from raw concepts to EDEN) — how we GOT HERE
- **`Plans/`** = detailed feature specs and archived analysis documents

## Quick orientation

Start with **CONCEPT.md** to understand EDEN's vision, then **PRD.md** for what we're building and in what order, then **RISKS-AND-GAPS.md** for what to avoid.
